# Conversion fidelity matrix

This document is the per-tag reference for what `@kleiobase/gedcom-converter` does when it converts between GEDCOM versions. It is the source of truth for whether a given structure round-trips cleanly, is degraded, or is dropped — and for which diagnostic code is emitted when that happens.

The matrix is generated by inspection of `src/mappings/551-to-v7.ts`, `src/mappings/v7-to-551.ts`, `src/mappings/date/*-to-*.ts`, the input-encoding layer in `src/utils/text.ts` + `src/utils/ansel.ts`, and the post-mapper sanitisation layer in `src/gedcom551/compatibility.ts`. The valid value sets for every enumeration (§3.4) live in `src/enums/index.ts` — the single source of truth shared by both mappers, with the `enumOrPhrase` helper driving the `OTHER` + `PHRASE` fallback. When this document and the code disagree, the code wins — please open an issue.

## Legend

| Outcome | Meaning |
| --- | --- |
| `clean` | Full semantic equivalence. No diagnostic emitted. |
| `lossy: <reason>` | Partial mapping — the target version cannot fully express the source. A diagnostic is emitted so callers can detect the degradation. |
| `_TAG` | Preserved as a `_`-prefixed custom extension. SCHMA declares it on the v7 side. |
| `dropped: <reason>` | Cannot be expressed in the target version. The structure is removed from the output. |
| `N/A` | The tag does not exist in the source version (e.g. `SNOTE` in 5.5.1). |

## Directions

- **v7 → 5.5.1**: GEDCOM 7.0.18 down-conversion. Combines the inverse mapper in `src/mappings/v7-to-551.ts` with the post-mapper sanitisation in `src/gedcom551/compatibility.ts` (which applies the bulk of 5.5.1-compliance rewrites).
- **5.5.1 → v7**: GEDCOM 5.5.1 up-conversion through `src/mappings/551-to-v7.ts`.
- **5.5 → v7**: Transitive — the converter does `5.5 → 5.5.1` (version bump only, no structural change) then `5.5.1 → v7`. The outcomes are therefore identical to the 5.5.1 → v7 column unless the 5.5 form differs lexically. Structures introduced in 5.5.1 (e.g. `EMAIL`, `FAX`) are treated as no-ops on the 5.5 side.

## How to read the tables

Each row is one tag. The "Notes" column flags context-specific behaviour (e.g. a tag behaves differently under `OBJE.FILE` than under `PLAC`). Multiple rows for the same tag mean the tag has materially different handlers in different parents.

## Pointer tags (general behaviour)

All pointer tags (`ALIA`, `ANCI`, `ASSO`, `CHIL`, `DESI`, `FAMC`, `FAMS`, `HUSB`, `NOTE`, `OBJE`, `REPO`, `SOUR`, `SUBM`, `WIFE`) share:

- v7 → 5.5.1: `clean` for pointer-to-existing-record; `lossy` (rewritten to a NOTE explaining what was referenced) when `@VOID@`. Diagnostic: `VOID_POINTER_NOTED`. Pointers to *missing* records are dropped with `DROPPED_MISSING_POINTER`.
- 5.5.1 → v7: `clean`.
- `HUSB` / `WIFE` / `CHIL` `PHRASE` children are hoisted into a per-pointer note on the inverse direction (`POINTER_PHRASE_NOTED`).

## Diagnostic codes (alphabetical)

Every code emitted by the converter, with the direction in which it can appear. `up` = 5.5.1 → v7, `down` = v7 → 5.5.1, `compat` = post-mapper sanitisation (down-only).

| Code | Direction | Meaning |
| --- | --- | --- |
| `AFN_TO_EXID` | up | 5.5.1 AFN → v7 EXID with legacy TYPE URI. |
| `AGE_PHRASE_FALLBACK` | up | Non-numeric AGE moved into a PHRASE substructure. |
| `AGE_PHRASE_NOTED` | compat | AGE PHRASE that doesn't fit 5.5.1 promoted to an event note. |
| `AGE_PHRASE_PROMOTED` | compat | Legacy `CHILD`/`INFANT`/`STILLBORN` PHRASE promoted into the AGE value. |
| `ASSO_PHRASE_NOTED` | compat | ASSO PHRASE hoisted to an association NOTE. |
| `ASSO_ROLE_SYNTHESIZED` | up | Synthesised `ROLE OTHER` when 5.5.1 source omits `RELA` (v7 requires ROLE). |
| `ASSOCIATION_NOTED` | compat | Unsupported ASSO in FAM/EVENT context rewritten as a note. |
| `AT_CONTINUATION_NOTED` | compat | NOTE continuation lines starting with `@` rewritten so 5.5.1 doesn't misread them as pointers. |
| `CALN_MEDI_PHRASE_NOTED` | compat | REPO CALN MEDI PHRASE hoisted to a citation NOTE. |
| `CITATION_EVENT_PHRASE_NOTED` | compat | `SOUR.EVEN.PHRASE` hoisted to a citation NOTE. |
| `CITATION_ROLE_PHRASE_FALLBACK` | up | `SOUR.EVEN.ROLE` free text → `ROLE OTHER` + `PHRASE`. |
| `CONTACT_ID_NOTED` | compat | `SKYPEID` / `JABBERID` hoisted to a note. |
| `CREA_NOTED` | compat | `_CREA` rewritten as a record note. |
| `CREA_TIME_NOTED` | compat | `_CREA` TIME appended to the creation note. |
| `DATE_CALENDAR_ESCAPE_CONVERTED` | up | 5.5.1 `@#D…@` calendar escape converted to v7 keyword form. |

## Open follow-ups

- [GED-11](https://linear.app/kleiobase/issue/GED-11) — full date round-trip. **Done:** interpreted `INT … (…)` and bare `(…)` phrases round-trip exactly (`DATE_INT_CONVERTED`, `DATE_PHRASE_EXTRACTED`); ranges (`BET…AND`/`AFT`/`BEF`), approximations (`ABT`/`CAL`/`EST`), periods (`FROM…TO` incl. open-ended), partial dates, dual dates (`1648/49`), mixed-calendar ranges (§6.3) and `BCE`/`B.C.` markers all round-trip valid→valid. The only lossy construct is the v7 PHRASE, which reconstructs via the 5.5.1 `INT`/`(…)` forms. Covered by `test/dates.test.ts`.
- [GED-12](https://linear.app/kleiobase/issue/GED-12) — Hebrew ADR/ADS leap-year resolution. **Done:** `isHebrewLeapYear` + `resolveHebrewAdar` (`src/mappings/date/hebrew.ts`) correct `ADR`→`ADS` in common years in both directions; covered by `test/hebrew-calendar.test.ts`.
- [GED-13](https://linear.app/kleiobase/issue/GED-13) — French Republican calendar. **Done:** all 13 month tags round-trip; invalid months and the illegal epoch marker are flagged (`src/mappings/date/calendar-validation.ts`, `test/french-republican.test.ts`).
- [GED-14](https://linear.app/kleiobase/issue/GED-14) — Julian epoch markers + legacy `ROMAN`/`UNKNOWN`. **Done:** `BCE`↔`B.C.` round-trips both ways; the undefined `ROMAN`/`UNKNOWN` calendars degrade to a PHRASE (up) / re-escape (down) with `UNKNOWN_CALENDAR` (`test/julian-legacy-calendar.test.ts`).
- [GED-15](https://linear.app/kleiobase/issue/GED-15) — bidirectional round-trip of every enum set. **Done:** the 12 enumeration sets are codified in `src/enums/index.ts` with the shared `enumOrPhrase` helper; both mappers consume them, PEDI now case-folds with `OTHER` + `PHRASE` fallback, and `test/enums.test.ts` exercises every set.
- [GED-16](https://linear.app/kleiobase/issue/GED-16) — character encoding. **Done:** input byte streams are decoded from ANSEL (pre-7.0 default, including combining diacritics), UTF-16 LE/BE (`UNICODE`), ASCII, or UTF-8 based on BOM + `1 CHAR` (`src/utils/ansel.ts`, `src/utils/text.ts`); the serializer takes a `lineEnding` option (LF/CRLF/CR); CR and LF inputs parse to identical IR (`test/encoding.test.ts`). Full BCP-47 ↔ language-name round-trip is tracked separately (only a handful of languages are aliased today).
- [GED-19](https://linear.app/kleiobase/issue/GED-19) — reduce remaining `_TAG` fallbacks. First pass done: `PLAC.MAP` and `PLAC.NOTE` now stay clean (both are native 5.5.1), and `PEDI OTHER` is surfaced as a `FAMC.NOTE`. Remaining deliberate `_TAG` keeps (`_LANG`, `_CROP`, `_TRAN`, `_EXID`, `_UID`, `_FILE`) have no cleaner 5.5.1 representation and are justified inline above.
